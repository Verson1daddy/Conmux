# 关于 Conflux

Conmux 是地基，**Conflux 是盖在它上面的那栋楼**。

如果说 conmux 解决的是"多个命令行会话怎么在 Windows 上跑得起来、看得住"，那 Conflux 想解决的是更上层、也更具体的一件事：**你同时开了三四个 AI 编程 agent，怎么才能不被它们淹没。**

> 一句话：**conmux 管面板和进程，Conflux 管 agent。** conmux 不知道什么叫"agent"——它只认面板、进程、字节；"这是一个 Claude Code 会话、它在等你批权限"这种语义，是 Conflux 干的活。

## 它想解决的问题

你在 Windows 上跑 Claude Code、Codex、Aider、OpenCode 这类 agent CLI。一个跑得挺好，三四个一起跑就没法看了——你不可能同时盯四个终端，一个个翻谁卡住了、谁在等你点确认、谁已经跑完。

Conflux 的设计目标就是让你**不必逐个盯守**：把"现在需要你的那个"浮出来，让你一键跳回它当初发问的那一刻，并在你没看的时候把发生的一切记进审计。

它**不是又一个聊天窗**。agent 的工作在这里被当成真实的会话、画布上的卡片、注意力信号、权限请求，以及一条可回放的事件时间线。

## 长什么样、能干什么

Conflux 是一个 **Tauri 2 + React 的 Windows 桌面工作台**，跑在 conmux 内核之上。核心是这么几件事：

- **多 agent 一块画布**——同时打开多个真实 CLI agent，每个都是活的 PTY 会话（ConPTY + xterm.js 渲染，完整 ANSI/配色），以卡片形式铺在同一块画布上，可排布、展开、收起。**不是假卡片、不是只读样稿。**
- **会来找你的注意力面**——灵动岛 + 侧栏 + 系统托盘，把每个 agent 的状态送到你眼前。对 Claude Code 会话，"这个在等你"来自一个权威 **hook** 信号，不是刮屏猜的——所以它是一个真实事件，而不是估计。
- **一键跳回触发处（jump-back）**——点通知，Conflux 直接把你带到那个分屏、带到触发它的那个位置，不用在一堆终端里翻找。
- **不打断心流地介入**——展开任意卡片进入双向终端，直接往那个 agent 的会话里打字，处理完再收回网格。
- **闸门在你手里（权限审批）**——agent 的权限请求以审批 UI 浮现，没经你放行的东西不会往下走。
- **一切留痕**——每个事件写入本地 **SQLite**，会话时间线可事后回放。

## 老实话：已经通的核心，和还在打磨的理念

这是这一页最该说清楚的地方。Conflux 里**真正跑通、稳定可用**的，是这么一条链：

> **多开真 CLI agent → 观测每个会话 → 注意力信号把该处理的浮出来 → 你审批 / 一键跳回 → 全程写进 SQLite 审计、可回放。**

这条"监管台"的链子是实的。但**agent 之间怎么协作、怎么编排，这套理念仍在打磨**，现在**没有做**的部分，我们直说：

- **讨论面板是单向广播，不是协作。** 它只做一件事：把你的一条 prompt **一次性广播**发给多个 agent。
- **agent 之间不互相对话。** 它们**不会**把回复汇进一个共享聊天室，**不会**彼此交谈——广播出去之后，各跑各的。
- **没有自动编排引擎。** 没有一个"自己决定该派谁干什么活"的调度大脑。**决策权始终在你手里**——Conflux 监管 agent，但不替你编排它们。

换句话说：**监管这一面是真打通的，协作 / 编排这一面还是个方向、不是现状。** 别把路线图当成已经能用的功能。

> 注意力面由**两态灵动岛 + 侧栏 + 系统托盘**构成；早期设计里的悬浮球（Float Ball）已经删掉，不在其中。

## 当前状态

- **V1 · 仅 Windows · early。** 它是一个**能用的工作台**，不是成品。
- 直接跑在 **ConPTY** 上，**不依赖 WSL**、不装类 Unix 兼容层。
- **暂无预编译 / 签名安装包**，目前请**从源码构建**（`git clone` → `conflux-app` 下 `npm install` + `npm run tauri:build`）。上线的安装包会是**未签名**的——没预算买代码签名证书，SmartScreen 会提示"发布者未知"，点「更多信息 → 仍要运行」即可。

路线上还在往前走的（roadmap，非现状）：预编译 / 签名安装包、更广的适配器覆盖与更简单的配置、注意力层与权限层的硬化、时间线 / 审计体验打磨。

## Conflux 和 Conmux 是什么关系

- **`conmux`**——独立的 Rust crate：Windows 终端多路复用 + agent 隔离运行时（ConPTY、进程整树监管、单一受审输入路径……），**不依赖 Tauri**。它是本手册的主角，也能单独用。
- **`conflux-app`**——盖在 `conmux` 之上的 Tauri 2 + React 产品层（画布、注意力面、权限 UI、审计 / 时间线）。

所以你现在读的这份手册，讲的是**地基 conmux**；Conflux 是这块地基**目前最大的一个消费者**，也是"mechanism（机制）vs semantics（语义）"这条边界为什么值得存在的活例子。想深入了解 Conflux 本身，去看它的项目仓库。
