# 这是什么、为什么值得用

*（这一页假设你没用过 tmux 或任何"终端复用器"。如果你用过，直接跳到 [从 tmux 过来](../from-tmux/differences.md)。）*

## 先说问题

你在命令行里跑东西——比如一个 AI 编程 agent（Claude Code、Codex…），或者一次编译、一个本地服务器。一个终端窗口跑一个。想同时跑三四个？那就得开三四个窗口，然后在它们之间来回点：一个个看谁卡住了、谁在等你输入、谁跑完了。

窗口一多，你就成了那个"人肉调度器"——不停切来切去，还生怕漏掉哪个正在等你。

## Conmux 做什么

Conmux 把多个命令行会话**装进一个窗口**，各占一块（叫「pane / 面板」）。在这个窗口里你可以：

- **分屏**——一个窗口切成几格，几个会话并排看。
- **切换**——一排小圆点代表每个会话，点一下就跳过去。
- **离开再回来**——关掉 Conmux 窗口，里面的会话**不会死**：它们在后台继续跑；下次打开，画面原样接回来（这叫 detach / attach）。
- **看得住**——哪个会话需要你（比如某个 agent 弹了权限请求），Conmux 会把它标出来，而不是让你自己一个个翻。

一句话：**它替你盯着那些命令行，你只在真正需要你的时候出手。**

## 为什么是 Conmux，而不是 tmux

如果你搜过"终端复用器"，多半会看到 **tmux**——Linux / macOS 上的经典工具。问题是：**tmux 不为 Windows 而生**。在 Windows 上用它，你得先装一层 WSL（一个跑在 Windows 里的 Linux 子系统）。

Conmux 直接跑在 Windows 自己的终端底层（叫 **ConPTY**）上，**不需要 WSL**。而如果你*想*把 WSL 里的工具接进来，它也能——[单独一节](../from-tmux/wsl.md) 讲这个。

它也比"把 agent 塞进一个完整 IDE（比如 VS Code 加一堆插件）"要轻：Conmux 围着命令行本身转，没有编辑器那一大坨。

> **诚实提醒**：带图形界面的 Conmux 用了系统自带的 WebView2，**不是**最极致的轻量方案（内存地板大约几百 MB）。它比一整套 IDE 省得多，但如果你追求"极致省内存"，纯命令行的 `conmux` 工具（不带图形界面）才是最轻的那档。
